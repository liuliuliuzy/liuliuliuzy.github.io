2、发布

第四步、配置适应多平台写作

Hexo是一个博客框架，你创建发布博文的过程，实际上是通过修改本地的Hexo项目，然后生成静态资源并部署来完成的。所以，当你更换一台电脑的时候，原本的博客项目该怎么保留呢。很显然，我们需要对博客源文件进行同步，方法当然是使用github。

1、在github上的博客仓库新建一个分支

以下操作最好在部署博客之前进行。

  • pull并且创建新分支

    2、在该分支上备份博客源码

  • 创建.gitignore文件
    因为博客源文件中有些是不需要备份的，比如node_modules、public等，所以需要创建该文件，告诉git这些东西不需要对其进行版本控制。
  • 将你的博客源文件进行备份
    在该分支上将你的源文件copy过来，然后git add\commit\push，将源文件推送到远程仓库
